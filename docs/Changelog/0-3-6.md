# Changelog — 0.3.1 → 0.3.6

---

## 0.3.1 — Bug fix: max attempts semantics

**`KeepRunningUntilSuccess` / `KeepRunningUntilFailure`**

- `max_attempts = 0` (and any negative value) now correctly means **unlimited** retries. Previously `0` would cause the node to exit immediately on the first failure/success.
- `KeepRunningUntilFailure`: when `max_attempts` is exhausted and the child never failed, the node now correctly returns `SUCCESS` (was incorrectly returning `FAILURE`).

---

## 0.3.2 — `FullAction` / `IFullActionConfig` naming alignment

The config keys used by `FullAction` and `CreateFullAction` were updated to match the node lifecycle method names introduced in 0.3.0:

| Old key         | New key             |
| --------------- | ------------------- |
| `OnFinish`      | `OnExit`            |
| `OnActivated`   | `OnBecameActivated` |
| `OnDeactivated` | `OnBecameInactive`  |

---

## 0.3.3 — New node: `WasEntryUpdated`

A leaf node that watches a list of blackboard keys and returns `SUCCESS` on any tick where at least one of their values has changed since the previous tick.

```typescript
new BTree.WasEntryUpdated(["health", "target"], /* skipFirst */ true);
```

- `entries` — array of blackboard key names to watch
- `skipFirst` (optional, default `false`) — when `true`, snapshots the current values on `OnBecameActivated` so the first tick does not fire as a false positive
- Resets its snapshot via `OnBecameInactive` when the node leaves the active set

---

## 0.3.4 — `BTCreator` versioned schemas

`BTCreator` now loads node creators based on the `version` field in the JSON data, resolved at `LoadData()` time.

| JSON `version` | Node set                                                                                            |
| -------------- | --------------------------------------------------------------------------------------------------- |
| `"1.0.0"`      | Default nodes + legacy `RetryUntilSuccess` / `RetryUntilFailure` aliases                            |
| `"2.0.0"`      | Default nodes + `KeepRunningUntilSuccess`, `KeepRunningUntilFailure`, `TryCatch`, `WasEntryUpdated` |

The methods `LoadDefaultNodes()`, `LoadV1Nodes()`, and `LoadV2Nodes()` are now **public** to allow subclasses or custom setups to compose node sets manually.

---

## 0.3.5 — Bug fix: `BehaviorTree.Halt()` incomplete teardown

`BehaviorTree.Halt()` previously only halted the root node. It now:

1. Calls `Halt()` on every node in `running_nodes_`
2. Calls `OnBecameInactive()` on every node in `active_nodes_`
3. Clears both sets

This ensures all in-flight nodes are properly cleaned up when the tree is manually stopped.

---

## 0.3.6 — New nodes: `Plug` and `OneShot`

### `Plug`

A stub leaf node that always returns a fixed status. Useful as a placeholder during development or in tests.

```typescript
new BTree.Plug(BTree.ENodeStatus.SUCCESS); // always SUCCESS (default)
new BTree.Plug(BTree.ENodeStatus.FAILURE);
new BTree.Plug(BTree.ENodeStatus.RUNNING);
```

### `OneShot`

A decorator that executes its child **once**, caches the result, and returns that same result on every subsequent tick without re-running the child.

```typescript
new BTree.OneShot(child, /* resetOnBecomeInactive */ false);
```

- While the child is `RUNNING`, `OneShot` passes through `RUNNING` and continues ticking the child.
- Once the child returns `SUCCESS` or `FAILURE`, that result is cached permanently.
- With `resetOnBecomeInactive = true`, the cache is cleared when the node leaves the active set, allowing the child to run again next time the node becomes active.

### `BehaviorTree` — Halt scheduling

`BehaviorTree.Halt()` can now be safely called from **inside a tick** (e.g. from within an `Action` callback). If the tree is currently mid-tick, the halt is deferred and executed automatically as soon as the tick completes, instead of corrupting the in-progress traversal.

### `BTCreator` additions

- `Plug` and `OneShot` added to the default node creator set (available in all schema versions).
- `"2.0.0"` schema: added `WasEntryUpdated` node creator (previously missing from v2 definition).
