# Changelog — 0.3.0

## BehaviorTree — Major Refactor

### Node lifecycle overhaul

- `OnFinish(status, bb)` replaced by three focused callbacks:
  - `OnExit(status, bb)` — called on every terminal result (including halt)
  - `OnSuccess(bb)` — called when the node returns `SUCCESS`
  - `OnFailure(bb)` — called when the node returns `FAILURE`
- `OnActivated(bb)` → **`OnBecameActivated(bb)`**
- `OnDeactivated(bb)` → **`OnBecameInactive(bb)`**
- `Halt()` now calls `OnExit` after `OnHalt`, so exit cleanup always runs

### `IFullActionConfig` / `FullAction` / `CreateFullAction`

- Config keys updated to match the new lifecycle names:
  - `OnFinish` → `OnExit`
  - `OnActivated` → `OnBecameActivated`
  - `OnDeactivated` → `OnBecameInactive`
- Added `OnSuccess` and `OnFailure` config keys

### Renamed nodes

- `RetryUntilSuccess` → **`KeepRunningUntilSuccess`**
- `RetryUntilFailure` → **`KeepRunningUntilFailure`**

### New node: `TryCatch`

A composite with 2–3 children: `[try, catch, finally?]`. Runs `try`; on `FAILURE` runs `catch`; always runs `finally` when present. The `finally` result is ignored — the outcome of `catch` (or `try` on success) is returned.

### Bug fixes

- `BehaviorTree.GetActiveNodes()` was incorrectly returning `running_nodes_`; now correctly returns `active_nodes_`
- `Parallel` now tracks already-completed children each tick to avoid re-ticking them
- `MemorySequence` resets its saved index via `OnBecameInactive` when the node leaves the active set
- `Condition` node no longer accepts a `Promise`-returning callback (sync only)

## FSM — New Features

### `ForceSetState(state)`

Immediately switches to the target state, bypassing all transition conditions. Safe to call from inside `Update()` — the change is scheduled and applied after the current tick completes.

### `AddAnyEventTransition(to, eventName, priority, condition?)`

Registers a global event transition that can fire from any active state (analogous to `AddAnyTransition` for condition transitions).

### `HandleEvent` improvements

Now also checks `any_event_transitions_` in addition to per-state event transitions.

### `Start()` assertion

Throws if the default state was never registered, catching misconfiguration early.
